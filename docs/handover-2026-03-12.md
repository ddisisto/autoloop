## Key design changes

### Gradient temperature ramps (replaces fixed T_survey/T_heat)

COOLING ramps T down by `dT_frac` (5%) per segment; HEATING ramps up. `T_min`/`T_max` are bounds. This finds basins at different accessibility temperatures rather than snapping to a fixed low T. Each capture records `T_capture` — the actual temperature when stabilization triggered.

- `SurveyState` has `current_T`, `cool()`, `heat()` methods (multiplicative steps with clamping)
- `SurveyController.decide()` manages T directly, overriding the StateMachine's placeholder controllers
- Run starts at `T_max`, first COOLING ramp descends from there
- CLI: `--T-min`/`--T-max` replace `--T-survey`/`--T-heat`
- Schema v3: `T_survey` column renamed to `T_capture` in `basin_captures`

### Decisions log removed

`.decisions.json` sidecar stripped from `experiment.py`, `controller.py`, `explorer.py`, `migrate_runs.py`. Rationale: parquet has per-step T/L/entropy already; learning signal requirements are undefined; rule-based controller decisions have no learning value. Explorer now uses parquet-derived events for all run types.

## Current state of survey

### What works
- State machine cycles: COOLING → CAPTURED → HEATING → TRANSIT → COOLING
- Gradient T ramps with floor/ceiling clamping
- Capture characterization: spectrum, embedding, novelty detection, attractor text
- Centroid catalogue: load/save, cosine matching, type registration
- Schema v3 migration (automatic, empty tables)
- HEATING escapes at T_max ceiling (new fix)

### What doesn't work well

**Premature capture.** Stability detection (3 consecutive segments with entropy delta < 0.1) fires on normal dynamics. In the 5k-step smoke test at L=8, captured at ent=3.83, β=0.80 — that's rich dynamics, not a basin. The system never actually collapsed.

**TRANSIT is wasteful.** Holds at high T for L tokens to "flush context" before cooling again. With gradient ramps, this delays the next cooling cycle unnecessarily. May not need to exist as a separate state.

**Re-capture of same basin.** No mechanism to avoid cooling back into the same basin. If you land in the same type twice in a row, should anneal more aggressively before the next attempt.

**Escape threshold is absolute.** `ESCAPE_ENTROPY_RISE = 1.0` works for deep basins (ent~2.0) but is unreachable for shallow captures (ent~3.8). The T_max ceiling fix prevents deadlock but doesn't solve the underlying calibration.
